②利用table创建表格：

③三种插入样式表的方法

内部样式表：作用范围局限于单个文档（利用style标签在html文件中定义）

*从外部，内部到内联，作用的范围越来越小。我感觉就像c语言的函数一样，外部样式表在外部定义可以实现更加简洁的风格，以及方便模块化(就是新建一个css文件把要使用的样式写在里面，在写html文件的时候在开头利用link链接。具体取用只要用id/class在标签里面定义就行了)

试了一下，在外联的css文件里面如果用”.”开头命名，那在html文件里面，要用class引用；如果用”#”开头命名，就要用id引用了

一般情况下，优先级如下：

内联样式 > 内部样式>外部样式> 浏览器默认样式

JavaScript 异步编程

异步是与同步相对的概念。同步按代码顺序执行，异步不按照代码顺序执行，异步的执行效率更高，因为主线程发射一个子线程来完成任务，子线程独立于主线程，即使出现阻塞也不会影响主线程的运行。再传统单线程编程中，主线程作为一个线程，不能够同时接受多方面的请求。所以，当一个事件没有结束时，界面将无法处理其他请求。因此我们常常用子线程来完成一些可能消耗时间足够长以至于被用户察觉的事情，比如读取一个大文件或者发出一个网络请求。

但是子线程有一个局限：一旦发射了以后就会与主线程失去同步，我们无法确定它的结束，如果结束之后需要处理一些事情，比如处理来自服务器的信息，我们是无法将它合并到主线程中去的。

为了解决这个问题，JavaScript 中的异步操作函数往往通过回调函数来实现异步任务的结果处理。回调函数就是一个函数，它是在我们启动一个异步任务的时候就告诉它：等你完成了这个任务之后要干什么。这样一来主线程几乎不用关心异步任务的状态了，他自己会善始善终。

JavaScript HTML DOM EventListener

即事件监听

向id为myBtn的标签添加事件click，即addEventListener() 方法用于向指定元素添加事件句柄。用 addEventListener() 方法时, JavaScript 从 HTML 标记中分离开来，可读性更强， 在没有控制HTML标记时也可以添加事件监听。

语法：

  • 第一个参数是事件的类型 (如 “click” 或 “mousedown”).

  • 第二个参数是事件触发后调用的函数。

  • 第三个参数是个布尔值用于描述事件是冒泡还是捕获。该参数是可选的。

    对于冒泡和捕获的描述：

    事件传递有两种方式：冒泡与捕获。

    事件传递定义了元素事件触发的顺序。 如果你将 <p> 元素插入到 <div> 元素中，用户点击 <p> 元素, 哪个元素的 “click” 事件先被触发呢？

    在 冒泡 中，内部元素的事件会先被触发，然后再触发外部元素，即： <p> 元素的点击事件先触发，然后会触发 <div> 元素的点击事件。

    在 捕获 中，外部元素的事件会先被触发，然后才会触发内部元素的事件，即： <div> 元素的点击事件先触发 ，然后再触发 <p> 元素的点击事件。

    addEventListener() 方法中的 “useCapture” 参数就是用来指定传递类型的，useCapture的默认值为 false, 即冒泡传递，当值为 true 时, 事件使用捕获传递。
