队列

队列queue

C++中，queue 是标准模板库（STL）中提供的一个容器，用于实现队列。queue 是一个先进先出（FIFO，First-In-First-Out）数据结构，即最早进入队列的元素最早被移除。

C++ 中 queue 的常用操作：

  1. **push()**：将元素添加到队列的尾部（入队）。
  2. **pop()**：移除队列头部的元素（出队）。
  3. **front()**：访问队列的第一个元素，但不移除它。
  4. **back()**：访问队列的最后一个元素。
  5. **empty()**：判断队列是否为空。
  6. **size()**：返回队列中元素的数量。

常见应用：

C++ 的 queue 非常适合用于任务调度、广度优先搜索（BFS）、缓存系统等需要先进先出的场景。

优先队列priority_queue

priority_queue 是 C++ 标准模板库（STL）中的一个容器适配器，用于实现优先队列。与普通的队列（queue）不同，优先队列在插入元素时会自动根据元素的优先级进行排序，保证每次取出的元素都是优先级最高的（在最大堆情况下）或优先级最低的（在最小堆情况下）。

priority_queue 的特点

  • 默认是最大堆：priority_queue 默认是实现为最大堆的数据结构，也就是说，优先级值较大的元素会被放在堆顶，优先出队。
  • 基于堆实现：priority_queue 的底层实现是使用了 堆（heap） 数据结构（通常是二叉堆），插入和删除操作的时间复杂度是 **O(log n)**。
  • 自动排序：当元素插入 priority_queue 时，它会自动按照优先级进行排序，保证最高优先级的元素在堆顶。

主要功能和接口

priority_queue 是一个容器适配器，提供了一些基本操作来管理元素：

  1. **push()**：向优先队列中添加新元素。
  2. **pop()**：移除优先队列中优先级最高的元素（即堆顶元素）。
  3. **top()**：访问优先队列中优先级最高的元素，但不移除它。
  4. **empty()**：判断优先队列是否为空。
  5. **size()**：返回优先队列中元素的个数。

默认行为：最大堆

默认情况下，priority_queue 是基于最大堆实现的，即优先级值较大的元素会排在堆顶。也就是说，优先队列中的元素是按从大到小排序的。

总结

  • priority_queue 是 C++ 中的一个容器适配器，用于实现优先队列，默认行为是基于最大堆实现的。
  • 可以通过 greater<T> 或自定义比较函数来实现最小堆或自定义排序。
  • priority_queue 在插入和删除操作上具有对数时间复杂度 O(log⁡n)O(\log n)O(logn)，非常适合用于优先级管理任务。

动态数组vector

通过这个头文件可以调用C++ 标准库中的一个非常常用的容器，类似于动态数组。

动态大小：不像普通数组大小固定，vector 可以根据需要动态增长或缩小。

连续内存分配：vector 使用连续的内存块来存储元素，因此支持快速的随机访问，和普通数组一样。

自动管理内存：vector 会自动处理内存分配和释放，因此用户不需要手动管理内存。

灵活操作：vector 支持在容器的任意位置插入和删除元素，尽管这些操作可能比随机访问慢一些。
